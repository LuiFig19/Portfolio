## Image Tips

### What Makes a Good Project Photo?

1. **Well-lit:** Use natural light or good indoor lighting
2. **Clean background:** Plain white, black, or neutral color
3. **Focus:** Sharp, clear focus on the device
4. **Context:** Show the device in use or deployed when possible
5. **Professional:** Clean PCBs, no clutter, organized

### Image Sources

**If you have real projects:**
- Photograph your actual builds
- Export PCB renders from KiCad (3D viewer)
- Take photos of deployed devices
- Screenshot web interfaces/dashboards

**If you need placeholders temporarily:**
- [Unsplash](https://unsplash.com/s/photos/electronics) - Free high-quality photos
- [Pexels](https://www.pexels.com/search/circuit%20board/) - Free stock photos
- [Pixabay](https://pixabay.com/images/search/pcb/) - Free images

**Best approach:** Mix of PCB renders + real device photos

### KiCad PCB Render Guide

1. Open your project in KiCad
2. Go to **View → 3D Viewer**
3. Adjust angle and lighting
4. **File → Export Current View as PNG**
5. Use 1200x900px or higher resolution
6. Compress with TinyPNG

### Photo Equipment Needed

- Smartphone camera (modern phones work great)
- Good lighting (window light or LED lamp)
- Plain background (white paper or cloth)
- Optional: Small tripod or steady surface

### Image Optimization

Before adding images to the site:

1. **Resize** to 1200x900px (or similar aspect ratio)
2. **Compress** using [TinyPNG](https://tinypng.com/)
3. **Rename** to match the required filenames exactly
4. **Test** by opening the website locally

### File Naming

**Important:** File names are case-sensitive and must match exactly:

✅ Correct:
- `project-astrarmm.jpg`
- `project-airquality.jpg`
- `project-motion.jpg`
- `project-network.jpg`

❌ Wrong:
- `Project-AstraRMM.jpg` (wrong capitalization)
- `project_astrarmm.jpg` (underscore instead of hyphen)
- `astrarmm.jpg` (missing prefix)
